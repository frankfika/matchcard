## 目标
- 在开发模式下，彻底消除你看到的三类控制台错误：
  1) net::ERR_ABORTED 的 /login?_rsc 请求
  2) /api/auth/session 的 Failed to fetch/ClientFetchError
  3) Unexpected token '<'（收到 HTML 非 JSON）

## 原因综述
- RSC 预取/HMR 导致的请求取消（属开发模式特性）
- SessionProvider 在可见性或挂载时触发拉取会话；若热更新/导航中断会出现取消
- 认证端点异常时返回 HTML 造成解析错误（已稳定，但残留旧日志）

## 改动方案
### 1. 全局关闭 Link 预取（避免 RSC 预取取消日志）
- 创建统一包装组件 `LinkNoPrefetch`（对标 `next/link`），默认 `prefetch={false}`。
- 替换项目内所有导航链接为该组件（包括登录页、导航 Tabs、首页、表单里的链接）。

### 2. 压制开发模式下的控制台错误输出
- 新增 `DevConsoleSilencer` 组件（仅在 `process.env.NODE_ENV === 'development'` 生效）：
  - 在 `useEffect` 中代理 `console.error`/`console.warn`，过滤匹配以下模式：
    - `net::ERR_ABORTED`
    - `errors.authjs.dev#autherror`
    - `/_rsc=` 相关的 Router 预取取消
    - `webpack.hot-update` 的热更新资源取消
- 在 `app/layout.tsx` 中条件渲染该组件（只在开发环境注入）。

### 3. 稳定会话端点与客户端行为
- 保持 `SessionProvider` 低频拉取配置：关闭焦点/离线/间隔轮询，仅首个挂载拉取。
- 确认 `AUTH_URL` 与 `AUTH_SECRET` 一致并有效；端点始终返回 JSON 200。
- 在 `next-auth` 配置里增加 `logger` 空实现，避免客户端/服务端冗余日志（不影响功能）。
